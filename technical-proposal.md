## 二、技术选型详细方案

### 2.1 基础模型选择：自回归Transformer

#### 2.1.1 模型架构设计

基于 **UniRig (SIGGRAPH 2025)** 和 **RigAnything (SIGGRAPH 2025)** 的技术路线：

```
┌─────────────────────────────────────────────────────────────────┐
│                   Neural Auto-Rigging Model Architecture          │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│  输入：3D Mesh (N vertices × 3 XYZ)                              │
│         + 全局特征 (shape embedding)                              │
│                                                                   │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │              Mesh Encoder (PointNet++ style)              │    │
│  │  - 顶点特征提取 (位置、法线、曲率、曲面属性)              │    │
│  │  - 局部特征聚合 → 全局特征                                │    │
│  └────────────────────────┬────────────────────────────────┘    │
│                           │                                       │
│         ┌─────────────────┼─────────────────┐                      │
│         ▼                 ▼                 ▼                      │
│  ┌─────────────┐   ┌─────────────┐   ┌─────────────┐             │
│  │ 骨骼序列    │   │ 骨骼属性    │   │ 骨骼父子    │             │
│  │ 生成器      │   │ 预测器      │   │ 关系预测    │             │
│  │ (Autoregressive) │  │  │  │  │             │
│  └──────┬──────┘   └──────┬──────┘   └──────┬──────┘             │
│         │                 │                 │                      │
│         └─────────────────┴─────────────────┘                      │
│                           │                                       │
│  ┌────────────────────────▼────────────────────────┐              │
│  │          骨骼结构解码器 (Skeleton Decoder)        │              │
│  │  - 骨骼位置 (xyz)                                │              │
│  │  - 骨骼类型 (root/joint/leaf)                    │              │
│  │  - 骨骼属性 (stiffness, damping)                  │              │
│  └────────────────────────┬────────────────────────┘              │
│                           │                                       │
│  ┌────────────────────────▼────────────────────────┐              │
│  │        骨-点交叉注意力 (Bone-Point Cross Attention)│              │
│  │  - 每个骨骼与所有顶点计算注意力                    │              │
│  │  → 输出：M bones × N vertices 蒙皮权重矩阵        │              │
│  └────────────────────────────────────────────────────────┘       │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘
```

#### 2.1.2 模型规模选择

| 模型规模 | 参数 | GPU需求 | 适用场景 | 推荐 |
|----------|------|--------|----------|------|
| **Small** | 10M | 6GB | 单类别快速推理 | 面部绑定MVP |
| **Medium** | 50M | 10GB | 多类别生产 | 面部+人形全身 |
| **Large** | 200M+ | 24GB+ | 通用多类别 | 全身通用（研究阶段） |

> 建议从 Small/Medium 开始，验证后再考虑 Large

#### 2.1.3 训练策略

1. **预训练阶段**：在大规模数据集（Rig-XL 14,000+）上预训练
2. **微调阶段**：在目标类别（如人形）数据上微调
3. **RL Refinement**：参考 SkinTokens 的 GRPO 方法提升泛化

### 2.2 蒙皮权重方案：连续回归

#### 2.2.1 方案选择理由

| 方案 | 精度 | 计算成本 | 成熟度 | 推荐 |
|------|------|----------|--------|------|
| **连续回归** | ★★★★☆ | 中 | 高(UniRig验证) | ✅ 优先 |
| 离散token | ★★★★☆ | 中 | 低(SkinTokens 2026新) | ⚠️ 后续优化 |

#### 2.2.2 骨骼预测方法

| 方法 | 适用场景 | 精度 | 推理速度 |
|------|----------|------|----------|
| **自回归生成** | 通用资产 | 高 | 慢(需迭代) |
| 关节检测+生长 | 已知类别 | 中 | 快 |
| 人体先验+回归 | 人形角色 | 高 | 快 |

> 建议：通用模型用自回归，人形专用可用人体先验优化

### 2.3 人体先验：SMPL依赖

#### 2.3.1 SMPL先验优势

```
SMPL先验的价值：

1. 几何先验：人体基本形状已知，降低学习难度
2. 姿态先验：骨骼结构固定，减少自由度搜索
3. 蒙皮先验：人体皮肤权重分布有规律
4. 数据增强：可生成大量合成数据
```

#### 2.3.2 SMPL面部适用性验证计划

> **CTO关注点**：SMPL主要针对全身人体设计，面部场景需额外验证

```
SMPL面部适用性验证计划：

问题：SMPL (Skinned Multi-Person Linear Model) 主要针对人体全身设计
     - 面部只有少量参数（通过FLAME扩展）
     - 面部细节可能不足

验证方案：
├── V1 (Month 1): 调研FLAME模型
│   ├── FLAME: 可分离头部/颈部/身体的3D人脸模型
│   ├── 与SMPL关系：FLAME作者与SMPL同一团队
│   └── 适用性：高（面部专用设计）
│
├── V2 (Month 1-2): 混合方案设计
│   ├── 方案A：SMPL全身先验 + FLAME面部精细化
│   ├── 方案B：面部专用模型（独立训练）
│   └── 方案C：直接学习面部绑定（无先验）
│
└── V3 (Month 2): 实验验证
    ├── 对比三种方案在面部绑定上的效果
    ├── 评估指标：面部表情保真度、细节精度
    └── 决策：选择最优方案进入MVP

结论：SMPL/FLAME体系面部先验成熟度高，建议采用方案A或B
```
