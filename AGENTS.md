# 每周游戏技术洞察 · 周报规范 (AGENTS.md)

本文件定义 **xhyha/insight** 仓库「每周游戏技术洞察」周报的标准化执行规范。
当用户说 **「更新本周游戏洞察」**（或等价表述）时，按本规范执行。

---

## 1. 触发与命名

- **触发短语**：`更新本周游戏洞察` / `本周游戏技术周报` / `weekly gaming insight` 等。
- **周次计算**：按 ISO 周。今天是 W{当前周}，覆盖 **本周一 ~ 今天**（一般 7 天窗口）。
- **文件命名**：`Weekly_Gaming_Insight_W{周次}_{年份}.pptx`
  - 例：`Weekly_Gaming_Insight_W25_2026.pptx`
- **存放位置**：仓库根目录 `/`（与历史报告同级）。
- **覆盖周期**：文件名周期 = 周一日期 ~ 当天日期，显示在封面与封底。

## 2. 数据采集（多源交叉核验）

固定优先源（英文业界权威）：
1. **Game Developer** (gamedeveloper.com) — 商业/行业/编程
2. **80 Level** (80.lv) — 美术/技术深读
3. **Game Developer / GDC** — 演讲与管线
4. **TechRadar / DSOGaming / Wccftech** — 引擎与硬件

补充源：GamesIndustry.biz、IGN、官方公告（开发博/新闻室）、厂商财报/社交媒体。

**核验要求（硬性）**：
- 每条要闻必须有 **≥1 个原始来源 URL** 且实际访问核对过内容。
- 数字、日期、人名、技术术语 **逐条核对原文**，不得凭记忆/推测。
- 若事实有歧义或与既有认知冲突，在周报中显式标注「⚠ 重要更正」。
- 用 `Task`(explore) 子代理并行抓取 + 核验，避免主线上下文过载。

## 3. 周报结构（白底主题，~20 页）

复用 `gen_weekly_w25.py` 的设计系统与 helper（见第 5 节）。章节骨架：

| 章节 | 内容 |
|------|------|
| 封面 (1) | 周次 / 主题标语 / 标签 / 数据源声明 |
| 导航 (1) | 四大板块 + 核验说明（含更正项） |
| 板块 01 引擎革新 | 本周引擎/图形/工具类要闻深读 |
| 板块 02 商业与变现 | 商业模式 / 销量 / 广告变现 |
| 板块 03 硬件与格局 | 平台 / 硬件 / 行业并购裁员 |
| 板块 04 趋势与启示 | 五大趋势综合 + 对中国/华为生态启示 |
| 下周看点 (1) | W{N+1} 关注清单 |
| 数据来源与核验 (1) | 原始来源表 + 更正声明 |
| 封底 (1) | 关于本周报 / 流程 / 规范指引 |

每个板块前用 `divider()` 大数字分隔页。每个板块 2~4 页。

## 4. 多轮内部审查（上传前必做）

在 `git push` 前完成以下检查，全部通过才上传：

1. **事实核验**：每条要闻已对原始来源核对（第 2 节）。
2. **边界检查**：运行形状边界脚本，确认 **0 个越界**（画布 13.333×7.5 in）：
   ```python
   # 见 gen_weekly_w25.py 同目录的检查片段
   for sl: for sh: 确认 0≤l, 0≤t, l+w≤W, t+h≤H
   ```
3. **文本溢出**：dump 每页文本，人工通读，确认无截断/错位。
4. **页码/页脚**：每页 `N/TOTAL` 与「每周游戏技术洞察 · W{周} {年} · xhyha/insight」。
5. **更正显式化**：任何与上一期或常识冲突之处，在导航页 + 来源页双标注。

## 5. 设计系统（白底，复用）

颜色与 helper 定义见 `gen_weekly_w25.py` 顶部（与 WWDC26 v2 一致）：
- 背景 `BG=FFFFFF`；卡片 `CARD=F7F9FC` / `CARD_D=EFF3F9`
- 文字 `INK=161B2E` / `TXT=374151` / `MID=6B7280` / `LITE=9AA5BC`
- 强调色：`TEAL / PURPLE / GOLD / BLUE / GREEN / ORANGE / RED / HUAWEI`
- helper：`slide / rect / txt / T / header / chip / kpi / divider / bullets / page_num`
- 画布：`13.333 × 7.5 in`，`slide_layouts[6]`（空白版式）

**生成方式**：python-pptx 程序化生成（环境无 libreoffice，不做渲染预览）。

## 6. 上交流程

1. 生成 `Weekly_Gaming_Insight_W{周}_{年}.pptx` 到仓库根目录。
2. 完成第 4 节全部审查。
3. `git add` 该文件 → `git commit`（中文信息，注明周次与页数）→ `git push origin main`。
4. commit 信息格式：`每周游戏技术洞察 W{周} - {页数}页 ({周覆盖日期})`。
5. 确认 push 成功后，向用户报告周次/页数/关键更正项。
